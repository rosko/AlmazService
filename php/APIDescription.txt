 Данный документ описывает API для доступа к системе управления ресурсами.
 
 1. Общее
 
 Доступ к ресурсам осуществляется используя правильно составленый к серверу GET запрос.
 В переданом URI должен быть корректно указан тип ресурса, тип возвращаемого результата,
 имя допустимого метода и необходимые параметры в соостветсвии с указанным методом.
 
 общий URI имеет следующий вид:
 
 http://domainname?controller=resource&type=audio&method=get&id=1234&resptype=json
 
 для удобства использования все URI должны быть нормализированы и иметь следующую структуру
 
 http://domainname/resource/audio/get/1234.json
 
 данный пример илюстрирует доступ к аудио ресурсу с id 1234. Результат будет JSON типе.
 
 2. Контролеры
 
 Контролер определяет метод работы с данными. На данный момент предполаегается 2 типа
 контролеров
     - resource
     - meta

     2.1. Resource
     
     Данный тип контролера указывает на то что доступ осуществляется к высокоуровневым объектам
     которые не имеют в базе данных отдельную таблицу или структуру. Такие объекты создаются на
     уровне выполнения программы и являются ЛОГИЧЕСКИМИ. К данному типу объектов относятся все
     высокоуровневые объекты:
         - аудио
         - видео
         - новости
         - изображения
         и т.д.
      
     2.2. Meta
     
     Данный тип контролека указывает на то что доступ осуществляется к непосредственно объектам
     базы данных. Например:
         - Resource
         - ResourceObject
         - ResourceMetaData
         и т.д.
     доступ к данному типу объектов должен быть ограничен.

 4. Метод
 
 Поддерживаемый набор методов
 
     - get
     - search
     - add
     - delete
     
     4.1. GET (HTTP GET)
     
     Возвращает один или множество объектов указаного типа. Допустимые параметры
     id - id объекта
     from - порядковый номер начала выборки
     count - количество
     
     должен быть указан как минимум 1 параметр, Если указан параметр id тогда остальные параметры
     игнорируются.
     
     4.2. SEARCH (HTTP GET)
     
     Осуществляет поиск ресурсов в соответствии указаным критериям
     
     4.3. ADD (HTTP POST)
     
     Создает новый ресурс в зависимости от переданных данных. Данные передаются методом POST.
     
     4.4. DELETE (HTTP GET/POST)
     
     Удаляет ресурс по его id. Данные могут передаваться как GET так и POST методом.
